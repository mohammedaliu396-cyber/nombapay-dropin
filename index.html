How it works

From zero to payments in three steps

NombaPay Drop-in abstracts away the complexity of Nomba's API into a dead-simple developer interface.

STEP 01

Add the script

Drop one script tag into your HTML. No npm, no bundler, no build step required.

<script src="nombapay-dropin.js"></script>
STEP 02

Deploy the proxy

Host the backend folder on Render. Add your Nomba credentials as environment variables — they never touch your frontend.

Root directory: backend
Start command: npm start
STEP 03

Call .open()

Configure with your backend URL and trigger on any button click. The full Nomba checkout modal appears — card, transfer, USSD.

NombaPayDropin.setup({...}).open()
Features

Built for developers who move fast

Everything you need to accept payments — nothing you don't.

Zero dependencies

Pure vanilla JavaScript IIFE. Works in any browser, on any stack, without a build step or package manager.

Security-first architecture

Credentials live only in your backend environment. The browser never sees your client_id or private key.

Multiple payment methods

Cards, bank transfers, and USSD — all through Nomba's hosted checkout. No extra config needed.

Token caching

The backend caches Nomba access tokens for up to 50 minutes, reducing API calls and speeding up checkout.

Event callbacks

onSuccess, onClose, and onError callbacks give you full control over the payment lifecycle in your app.

Drop-in UX

Smooth modal with blur overlay, close on Escape, click-outside dismiss, and loading states built in.

Security

Your credentials. Your control.

NombaPay Drop-in is built around a core principle: sensitive credentials never touch the browser.

Backend proxy pattern

Your Nomba client_id and private_key live only in server environment variables. The plugin only ever receives a checkout URL.

Short-lived tokens

OAuth2 access tokens are generated server-side, cached for 50 minutes, and never forwarded to the client.

Gitignored secrets

.env files are blocked from Git commits. The repo only contains a .env.example showing structure, never real values.

Server-side verification

Payment status is verified through the backend before your app treats a transaction as complete.
